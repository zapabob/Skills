# Perplexity DeepResearch (脚注の裏取り) スキル

このスキルは、文書の脚注、参照元、または主張の事実確認（ファクトチェック）を徹底的に行い、情報の正確性、最新性、および一次情報との整合性を検証するためのガイドラインです。まるでPerplexityのDeep Research機能のように、自律的に深く検索して裏付けを取ることを目的とします。

## 目的

提供された脚注や主張が、信頼できるソースに基づいているか、文脈が正確か、最新の事実と一致しているかを検証し、ハルシネーションや不正確な情報を防ぎます。

## 実行手順

1. **検証対象の特定と分解**:
   - ユーザーから提供された文書、または指定された脚注・参照情報・主張を抽出します。
   - 複雑な主張は、検証可能な複数の小さな事実（ファクト）に分解します。
   - それぞれの事実に対して、「何を検証すべきか（Fact-checking target）」を明確にします。

2. **深い情報検索 (Deep Research) の実行**:
   - 利用可能なWeb検索機能（Antigravityの `search_web`、Claude CodeのMCP Web検索、Cursorの `@Web` など）を使用し、検証対象に関する情報を複数回、多角的に検索します。
   - **一次情報源の探索**: 単なるまとめサイトではなく、公式ドキュメント、学術論文、公式サイト、政府機関、信頼できる報道機関などを優先して探します。
   - **多角的なクエリ**:
     - 事実確認クエリ: `[キーワード] 公式発表`, `[キーワード] 論文`
     - 批判的反証クエリ: `[キーワード] 誤り`, `[キーワード] ファクトチェック`, `[主張内容] is false`
   - 必要に応じて各ツールのURL読み込み機能を用い、検索結果のURLからページ内の実際のテキストを読み取り、詳細なコンテキストを確認します。

3. **情報の照合と分析**:
   - **✅ 一致**: 脚注の主張が一次情報と完全に一致しているか。
   - **⚠️ 部分的一致 / ニュアンスの欠落**: 主張は概ね正しいが、重要な文脈（例えば「特定の条件下のみ有効」など）が抜け落ちていないか。曲解されていないか。
   - **❌ 矛盾 / 誤り**: 脚注の主張が最新の事実や一次情報と明確に矛盾している。
   - **❓ 出所不明**: 信頼できる情報源が見つからない、または信頼性が非常に低いソース（匿名のブログなど）にしか記載がない。

4. **検証レポートの作成**:
   - 検証した脚注・主張ごとに以下の形式でレポートをまとめ、ユーザーに提示またはドキュメントに反映します：
     - **対象の脚注/主張**: (元のテキスト)
     - **判定**: [ ✅ 正確 | ⚠️ 要注意 / 補足必要 | ❌ 誤り | ❓ 情報源不明 ]
     - **検証結果の詳細**: なぜその判定になったか、見つかった証拠は何か。
     - **情報源 (エビデンス)**: 参考にした信頼できるURLリスト。
     - **修正提案 (必要な場合)**: 不正確な情報を修正するための具体的な代替テキストや、追加・修正すべき参照元。

## ベストプラクティス (全エージェント共通: Antigravity / Claude Code / Cursor)

- **多段検索（Chain of Search）**: 最初に見つかった情報だけで結論付けず、その情報源がさらにどこを参照しているかを追いかけます。一次情報に辿り着くまで深掘りしてください。
- **時系列の確認**: 特に統計データ、法律、ソフトウェアのバージョンや仕様に関する脚注は、「いつの情報か」を特定します。より新しい情報が存在しないか確認してください。
- **LLMの内部知識に依存しない**: ハルシネーションの排除が最優先です。自身の知識に頼るのではなく、**必ず**利用可能なWeb検索ツールで得た外部証拠に基づいて判定してください。証拠が見つからない場合は「確証が得られない」と正確に報告してください。

### エージェント別 ツール活用ガイド

- **Antigravity**: `search_web` ツールで複数回検索し、さらに `read_url_content` を活用して一次ソースの本文を確認します。
- **Claude Code**: MCP経由のWeb検索サーバーや、自身に備わった関数/ブラウザ機能などを通じて外部の一次情報へとアクセスします。
- **Cursor**: ComposerにてWebアクセスを許可するか、`@Web` 機能を用いて検索結果を取得します。関連する参照ファイルの読み込みも並行して行ってください。
